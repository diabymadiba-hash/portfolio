Sophie Bluel — Portfolio d'architecte

Développement du site portfolio dynamique d'une architecte d'intérieur : manipulation du DOM, consommation d'API REST, authentification et gestion des œuvres via une modale.

Voir le site live GitHub
🏛️

Contexte du projet

Sophie Bluel est un projet de la formation OpenClassrooms centré sur la programmation JavaScript côté client. J'ai pris en main une base de code HTML/CSS existante pour y ajouter toute la dynamique : récupération des projets depuis une API, filtrage par catégorie, page de connexion sécurisée et interface d'administration permettant d'ajouter ou supprimer des œuvres sans recharger la page.

Objectifs

🎯Manipuler le DOM dynamiquement avec JavaScript
🎯Consommer une API REST avec Fetch (GET, POST, DELETE)
🎯Implémenter une page de connexion avec JWT
🎯Créer une modale d'administration fonctionnelle
🎯Gérer les événements et la navigation sans rechargement
🎯Persister le token dans le localStorage

Compétences développées

✓Manipulation avancée du DOM (createElement, appendChild, innerHTML)
✓Fetch API : requêtes asynchrones avec async/await
✓Gestion de l'authentification et des tokens JWT côté client
✓Création d'une modale avec upload d'image (FormData)
✓Filtrage dynamique de contenu par catégorie
✓Gestion des erreurs et retours utilisateur

Résultats

Le site affiche dynamiquement les projets depuis l'API, les filtre par catégorie en temps réel, et propose une interface administrateur complète après connexion. L'ajout et la suppression d'œuvres sont répercutés instantanément sans rechargement de page. Ce projet a été le déclic pour ma maîtrise de JavaScript asynchrone.

Stack technique
🟨 JavaScript ES6+
🔗 Fetch API
📄 DOM API
🔐 JWT (localStorage)
📤 FormData (upload)
🌐 HTML5 / CSS3
